# Swagger UI Integration

## Overview

This document describes the integration of Swagger UI React into the Studio canvas view, providing an interactive API documentation interface alongside the existing Canvas and Code views.

### Key Features

#### 1. Three View Modes
The Studio now supports three distinct views:

1. **Canvas View** - Visual class diagram with drag-and-drop functionality
2. **Code View** - Monaco editor showing OpenAPI 3.1.0 specification in JSON/YAML
3. **Swagger View** - Interactive Swagger UI for API exploration and testing

#### 2. Swagger UI Configuration

The Swagger UI component is configured with the following options:

```typescript
<SwaggerUI
  spec={JSON.parse(openApiSpec)}
  docExpansion="list"
  defaultModelsExpandDepth={1}
  defaultModelExpandDepth={3}
  displayRequestDuration={true}
  filter={true}
  showExtensions={true}
  showCommonExtensions={true}
  tryItOutEnabled={true}
/>
```

**Configuration Details**:
- `spec`: Uses the OpenAPI specification generated from the "Code" view
- `docExpansion="list"`: Expands only tags by default
- `defaultModelsExpandDepth={1}`: Shows one level of model nesting
- `defaultModelExpandDepth={3}`: Expands models up to 3 levels deep
- `displayRequestDuration={true}`: Shows how long API requests take
- `filter={true}`: Enables filtering/search functionality
- `showExtensions={true}`: Displays OpenAPI extensions
- `showCommonExtensions={true}`: Shows common vendor extensions
- `tryItOutEnabled={true}`: Allows users to test API endpoints directly

#### 3. Swagger View Features

The Swagger view includes:

- **Header Section**:
  - Title and description
  - Project and version information
  - Copy Spec button (copies OpenAPI JSON to clipboard)
  - Export Spec button (downloads OpenAPI specification as JSON file)

- **Main Content**:
  - Interactive Swagger UI interface
  - Expandable API endpoints
  - Request/response schemas
  - Try-it-out functionality
  - Model definitions
  - Search/filter capabilities

- **Empty State**:
  - Displays friendly message when no OpenAPI specification is available
  - Prompts user to add classes and properties

#### 4. Dynamic Import for SSR Compatibility

Both Monaco Editor and Swagger UI are dynamically imported to prevent server-side rendering issues:

```typescript
const SwaggerUI = dynamic(() => import('swagger-ui-react'), {
  ssr: false,
  loading: () => (
    <div className="h-full flex items-center justify-center">
      <div className="text-gray-500 dark:text-gray-400">Loading Swagger UI...</div>
    </div>
  ),
});
```

#### OpenAPI Specification Source

The Swagger UI view uses the same OpenAPI specification generated by the `generateOpenApiSpec()` utility function that's used in the Code view. This ensures consistency across all three views.

### User Experience

#### Switching Between Views

Users can easily switch between the three views using the view switcher buttons in the header:

1. **Canvas** - Visual diagram view
2. **Code** - Text-based OpenAPI specification
3. **Swagger** - Interactive API documentation

The view switcher is only visible when a project and version are selected.

#### Use Cases

- **Canvas View**: Best for designing and visualizing class relationships
- **Code View**: Best for viewing/copying the raw OpenAPI specification
- **Swagger View**: Best for exploring API endpoints and testing requests

### Benefits

1. **Interactive Documentation**: Provides a user-friendly interface for API exploration
2. **Testing Capability**: Allows users to test API endpoints directly from the UI
3. **Professional Presentation**: Uses industry-standard Swagger UI for familiar experience
4. **Consistency**: All three views use the same underlying OpenAPI specification
5. **No Configuration Required**: Automatically generates documentation from class definitions

To test the Swagger UI integration:

1. Navigate to the Studio page (`/ade/studio`)
2. Select a project and version
3. Click the "Swagger" button in the view switcher
4. Verify that:
   - Swagger UI loads correctly
   - API endpoints are displayed
   - Models/schemas are visible
   - "Try it out" functionality works
   - Copy and Export buttons function properly
   - Empty state displays when no classes exist
