# TypeScript DTO Generation - Quick Start Guide

## Overview
Generate TypeScript interfaces and types from your Apiome data models with a single click.

## How to Use

### 1. Access the Generate Tab
1. Open your project in Apiome Studio
2. Select a version
3. Click the **"Generate"** button in the top toolbar

### 2. Select TypeScript
1. In the Generate tab header, find the language dropdown
2. Click it and select **"TypeScript"**
3. The code will instantly update to show TypeScript interfaces

### 3. Review Generated Code
The generated code includes:
- TypeScript interfaces for each class
- Nested interfaces for complex properties
- Union types for enums
- Composition types (allOf/oneOf/anyOf)
- JSDoc comments with descriptions and constraints

### 4. Export or Copy
**Copy to Clipboard**:
- Click the **"Copy"** button
- Paste into your TypeScript project

**Export to File**:
- Click the **"Export"** button
- File will download as `schema.ts`
- Ready to import in your project

## Example Output

### Basic Interface

Given this data model:
```
User
  - id: string (uuid, required)
  - email: string (email format, required)
  - name: string (optional)
  - age: integer (0-150, optional)
  - roles: string[] (optional)
```

You get this TypeScript:
```typescript
/**
 * My Project
 * Version: 1.0.0
 *
 * Data Type Objects for My Project
 *
 * Generated by Apiome Studio
 */

/**
 * User entity
 */
export interface User {
  /**
   * Unique identifier
   * @constraints format: uuid
   */
  id: string;
  
  /**
   * User email address
   * @constraints format: email
   */
  email: string;
  
  /** Display name */
  name?: string;
  
  /**
   * User age
   * @constraints minimum: 0, maximum: 150
   */
  age?: number;
  
  /** User roles */
  roles?: string[];
}

// Export all types
export type AllTypes = {
  User: User;
};
```

### Inline/Nested Properties

Given this data model with inline object:
```
Customer
  - id: string (uuid, required)
  - name: string (required)
  - address: object (required)
    - street: string (required)
    - city: string (required)
    - zipCode: string (optional)
```

You get this TypeScript:
```typescript
/**
 * Address details
 */
export interface Address {
  street: string;
  city: string;
  zipCode?: string;
}

/**
 * Customer entity
 */
export interface Customer {
  id: string;
  name: string;
  address: Address;
}
```

### Arrays with Inline Objects

Given this data model with array of inline objects:
```
Order
  - id: string (uuid, required)
  - items: array (required)
    - productId: $ref Product (required)
    - quantity: integer (required)
    - price: number (optional)
```

You get this TypeScript:
```typescript
/**
 * Item in order
 */
export interface ItemItem {
  productId: Product;
  quantity: number;
  price?: number;
}

/**
 * Order entity
 */
export interface Order {
  id: string;
  items: ItemItem[];
}
```

**Note**: The naming follows Python's pattern:
- Object properties: PascalCase of property name (e.g., `Address` for `address`)
- Array items: Singular + "Item" suffix (e.g., `ItemItem` for `items[]`)

## Integration with TypeScript Projects

### Option 1: Direct Import
```typescript
// Copy schema.ts to your project's types/ folder
import { User, Product, Order } from './types/schema';

// Use in your code
const user: User = {
  id: '123e4567-e89b-12d3-a456-426614174000',
  email: 'user@example.com',
  name: 'John Doe'
};
```

### Option 2: Module Declaration
```typescript
// types/apiome.d.ts
declare module '@apiome/types' {
  // Paste generated code here
}

// Use in your code
import type { User } from '@apiome/types';
```

### Option 3: Shared Package
```typescript
// 1. Create npm package with generated types
// 2. Publish to npm or private registry
// 3. Install in all projects: npm install @yourorg/api-types
import { User, Product } from '@yourorg/api-types';
```

## Features

### Supported Type Mappings
- `string` → `string`
- `integer`/`number` → `number`
- `boolean` → `boolean`
- `array` → `Type[]`
- `object` → Nested interface or `Record<string, any>`
- `enum` → Union type: `"value1" | "value2"`
- `$ref` → Referenced type name

### Special Formats
- `uuid` → `string` (with format constraint in comment)
- `email` → `string` (with format constraint)
- `date` → `string` (with format constraint)
- `date-time` → `string` (with format constraint)
- `uri`/`url` → `string` (with format constraint)

### Composition Types
- `allOf` → Intersection: `Type1 & Type2 & Type3`
- `oneOf` → Union: `Type1 | Type2 | Type3`
- `anyOf` → Union: `Type1 | Type2 | Type3`

### Optional Properties
Properties are marked with `?` unless:
- Listed in the schema's `required` array

## Tips & Tricks

### Tip 1: Use with React
```typescript
import { User } from './types/schema';
import { FC } from 'react';

interface UserCardProps {
  user: User;
}

const UserCard: FC<UserCardProps> = ({ user }) => {
  return <div>{user.name || user.email}</div>;
};
```

### Tip 2: Use with React Query
```typescript
import { useQuery } from '@tanstack/react-query';
import { User } from './types/schema';

const useUser = (id: string) => {
  return useQuery<User>({
    queryKey: ['user', id],
    queryFn: () => fetch(`/api/users/${id}`).then(r => r.json())
  });
};
```

### Tip 3: Use with Axios
```typescript
import axios from 'axios';
import { User } from './types/schema';

const api = axios.create({ baseURL: '/api' });

const getUser = (id: string) => {
  return api.get<User>(`/users/${id}`);
};
```

### Tip 4: Use with Zod (Runtime Validation)
```typescript
import { z } from 'zod';
import type { User } from './types/schema';

// Create runtime validator from type
const UserSchema = z.object({
  id: z.string().uuid(),
  email: z.string().email(),
  name: z.string().optional(),
  age: z.number().int().min(0).max(150).optional(),
  roles: z.array(z.string()).optional()
}) satisfies z.ZodType<User>;

// Use for validation
const validateUser = (data: unknown): User => {
  return UserSchema.parse(data);
};
```

## Workflow

### Best Practice Workflow
1. **Design** - Create data models in Apiome Studio
2. **Generate** - Export TypeScript types
3. **Commit** - Add to version control
4. **Share** - Use across frontend projects
5. **Update** - Regenerate when models change
6. **Sync** - Keep backend (Python) and frontend (TypeScript) in sync

### Continuous Integration
```bash
# CI script to regenerate types
#!/bin/bash

# 1. Fetch latest from Apiome API
curl -X GET "https://your-apiome.app/api/generate/typescript" \
  -H "Authorization: Bearer $API_KEY" \
  -o src/types/schema.ts

# 2. Check for changes
if git diff --quiet src/types/schema.ts; then
  echo "No changes to types"
else
  echo "Types updated"
  git add src/types/schema.ts
  git commit -m "Update generated types from Apiome"
fi
```

## Troubleshooting

### Issue: Types don't match API responses
**Solution**: Regenerate types after any schema changes in Apiome

### Issue: Circular references
**Solution**: TypeScript handles these automatically with interfaces

### Issue: Missing properties
**Solution**: Check if properties are hidden or belong to nested objects

### Issue: Wrong types
**Solution**: Verify property types in Apiome Studio match your needs
