# TypeScript DTO Generation Feature

## Overview
Added TypeScript DTO generation support to the Studio's Generate tab, allowing users to export their data models as TypeScript interfaces and types alongside the existing Python/Pydantic option.

## Implementation Date
December 7, 2025

## Features

### TypeScript DTO Generator (`typescript-dto.ts`)
A new utility module that generates TypeScript DTOs from class definitions with support for:

- **Interfaces and Types**: Generates proper TypeScript interfaces for classes
- **Type Mappings**: Converts JSON Schema types to TypeScript types
  - `string` → `string`
  - `integer`/`number` → `number`
  - `boolean` → `boolean`
  - `null` → `null`
  - `array` → `Type[]`
  - `object` → `Record<string, any>` or nested interface
- **Enumerations**: Union types for enum values (e.g., `"value1" | "value2"`)
- **References**: Properly references other class types using `$ref`
- **Nested Objects**: Generates separate interfaces for nested object properties
- **Composition Support**: Handles JSON Schema composition keywords:
  - `allOf` → Intersection types (`Type1 & Type2`)
  - `oneOf`/`anyOf` → Union types (`Type1 | Type2`)
- **Optional Properties**: Marks properties as optional based on `required` array in schema
- **Property Comments**: JSDoc comments with descriptions and constraints
- **Constraints Documentation**: Documents validation constraints in comments:
  - Pattern, minLength, maxLength
  - Minimum, maximum
  - minItems, maxItems
  - Format specifications

### Studio UI Updates

#### Language Selector
- Added a dropdown in the Generate tab to switch between Python and TypeScript
- Located next to the title in the Generate view header
- Options: "Python" and "TypeScript"

#### Code Generation
- Both Python and TypeScript DTOs are generated simultaneously
- Cached separately to avoid regeneration when switching languages
- Updates automatically when:
  - Language selection changes
  - Classes are added/edited/deleted
  - View mode switches to Generate tab

#### File Export
- Download button uses correct file extension:
  - Python: `.py` (schema.py)
  - TypeScript: `.ts` (schema.ts)
- Appropriate MIME type for text files

#### Monaco Editor
- Syntax highlighting switches based on selected language
- Default placeholder text adapts to language (Python comments vs TypeScript comments)

## Usage

### In the Studio
1. Open a project and version in the Studio
2. Click the "Generate" tab
3. Use the language dropdown to select "TypeScript"
4. View the generated TypeScript interfaces
5. Copy to clipboard or export to a `.ts` file

### Generated TypeScript Format
```typescript
/**
 * Project Name
 * Version: 1.0.0
 *
 * Description text
 *
 * Generated by Objectified Studio
 */

/** Description of the class */
export interface ClassName {
  /**
   * Property description
   * @constraints format: email, minLength: 5
   */
  propertyName?: string;
  anotherProperty: number;
}

// Nested interfaces
export interface ClassNameNestedObject {
  nestedField?: string;
}

// Composition types
export type ComposedType = Type1 & Type2;  // allOf
export type UnionType = Type1 | Type2;      // oneOf/anyOf

// Export all types
export type AllTypes = {
  ClassName: ClassName;
  ComposedType: ComposedType;
};
```

## Benefits

1. **Multi-Language Support**: Users can now generate DTOs in both Python and TypeScript
2. **Frontend Integration**: TypeScript DTOs are ideal for TypeScript/JavaScript frontends
3. **Type Safety**: Generated interfaces provide compile-time type checking
4. **Documentation**: JSDoc comments include descriptions and validation constraints
5. **Efficient Switching**: Cached versions allow instant switching between languages
6. **Consistent API**: Same class definitions generate equivalent structures in both languages

## Testing

To test the feature:
1. Create a project with multiple classes
2. Add properties with various types (strings, numbers, arrays, objects)
3. Add nested properties
4. Use composition (allOf/oneOf/anyOf)
5. Switch between Python and TypeScript in the Generate tab
6. Verify generated code is syntactically correct
7. Export and test compilation in a TypeScript project

## Notes

- The TypeScript generator follows the same patterns as the Python generator for consistency
- Both generators are called on every data change to keep both cached versions up to date
- The feature integrates seamlessly with existing Studio functionality
- No breaking changes to existing Python generation
