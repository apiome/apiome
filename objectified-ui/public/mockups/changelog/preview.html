# Payments API — v2.4.1 Release Notes
_Released 2026-05-10 · Diff: v2.3.0 → v2.4.1_

---

## ⚠️ Breaking Changes

> These changes require consumer action before upgrading.

- **REMOVED** `DELETE /payments/{id}` — Use `POST /payments/{id}/cancel` instead.
  Migration: Replace all DELETE calls. Cancellation now records a reason and timestamps.

- **TYPE CHANGE** `PaymentIntent.amount` — Changed from `float` to `integer` (minor
  currency units). Example: `49.99` → `4999`. Update all request bodies.

- **REQUIRED** `POST /payments` — `currency` field is now required.
  Requests omitting it receive `422 Unprocessable Entity`.

- **RENAMED** `PaymentStatus` → `PaymentState` enum class.
  Update all enum references in consumer code.

---

## ✨ New Features

- Added `POST /payments/{id}/cancel` — explicit cancellation with audit trail.
- Added `POST /payments/{id}/dispute` — file a chargeback dispute.
- Added Payment Methods API: `GET/POST /payment-methods`,
  `DELETE /payment-methods/{id}`.
- Added `POST /webhooks/test` — trigger test webhook events from the dashboard.
- Added `GET /reports/settlements` and `GET /reports/fee-summary`.
- `PaymentIntent.metadata` — arbitrary key-value bag for consumer annotations.
- `PaymentIntent.capture_method` — control auto vs. manual capture.

---

## ⏳ Deprecations

> Fields marked deprecated will be removed in **v3.0.0**.

- `PaymentIntent.customer_id` — use `customer.id` nested object.
- `PaymentIntent.idempotency_key` — use `Idempotency-Key` HTTP header.
- `Refund.description` — use `notes` field instead.
