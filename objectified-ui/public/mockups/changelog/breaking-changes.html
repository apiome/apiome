Breaking Changes

v2.3.0 → v2.4.1 · 4 breaking changes

Requires consumer action
REMOVED DELETE /payments/{id} path_operation

The DELETE /payments/{'{id}'} endpoint has been removed. Payment cancellation is now performed via POST /payments/{'{id}'}/cancel to provide a richer audit trail.

⚠ Migration required: Replace DELETE calls with POST /payments/{'{id}'}/cancel before upgrading.

TYPE CHANGE PaymentIntent.amount class_properties

The amount field type changed from float to integer (minor currency units). This avoids floating-point precision issues in financial calculations.

- "amount": 49.99
+ "amount": 4999
REQUIRED FIELD POST /payments — currency path_operation

The currency request field is now required (was optional with default "USD"). Requests omitting this field will receive 422 Unprocessable Entity.

RENAMED PaymentStatus → PaymentState classes

The PaymentStatus enum class has been renamed to PaymentState for consistency with the event streaming model. All references in request/response schemas are updated.
