Release notes

Authored by the publisher. Markdown supported.

Highlights

This release lays the groundwork for multi-region tax handling and unblocks the upcoming Stripe + Adyen instrument split. It is fully backward-compatible at the consumer level — existing v2.4.x clients keep working until they choose to opt in.

Added

  • InvoiceLineItem — first-class line items with their own taxonomy, tax bracket, and discount references.
  • TaxBracket — region-aware tax brackets with an effective-from/to window.
  • PaymentInstrument — granular instrument metadata (network, last-four, fingerprint).

Changed

  • Invoice.lineItems is now a reference to InvoiceLineItem instead of an inline structure. Old shape is still accepted on read.
  • Payment.method split into Payment.provider + Payment.instrument. The legacy method field is computed and remains available.
  • Currency.code is now strictly validated as ISO 4217.

Notes

No fields were removed in this release. Two minor lint warnings remain — see the lint card above.
