Payments API

v2.4.1 REST · JSON

The Payments API provides a complete set of endpoints to create, capture, and manage payment transactions, refunds, and disputes. Designed for PCI-compliant integrations with support for card-not-present flows, pre-authorization holds, and multi-currency settlements.

🚀 Quick Start

# Install the SDK
npm install @objectified/payments-sdk

# Create your first payment
import { PaymentsClient } from '@objectified/payments-sdk';

const client = new PaymentsClient({ apiKey: 'YOUR_API_KEY' });
const payment = await client.payments.create({
  amount: 4999, currency: 'USD', customerId: 'cus_01HXYZ'
});

🔐 Authentication

All requests require a Bearer token in the Authorization header. API keys are scoped per environment — use test-mode keys for development and live keys for production.

Authorization: Bearer sk_live_XXXXXXXXXXXXXXXX
