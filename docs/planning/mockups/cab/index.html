CAB mockups

Visual designs for Apiome CAB — a structured, multi-stage approval workflow for publishing API changes. Design review → security review → architecture board sign-off → legal review (for external APIs) → release gate. Each stage carries assigned reviewers, SLA deadlines, inline comment threads, and a tamper-evident override audit log. Complements the diff and contracts mockups by adding the human governance layer required by enterprises with ITIL/change management processes. Mockups follow the Apiome Studio design system: Inter typography, violet/rose accent, slate gray scale, panel-based layout, and dark mode parity.
