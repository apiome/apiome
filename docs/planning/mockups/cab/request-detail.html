Payment API v3.2 — Add metadata field to POST /payments

Opened by Sarah Rodriguez · 2 days ago · Non-breaking change

Design Approved
Security In progress
Arch Board Pending

Change Description

Add an optional metadata field (type: object, max 4KB) to the POST /payments request body. The field enables consumers to attach arbitrary key-value pairs to payment records for downstream reconciliation. Field is fully optional; existing consumers are unaffected.
