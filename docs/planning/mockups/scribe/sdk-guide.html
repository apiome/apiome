Operations

  • POST /payments
  • GET /payments
  • GET /payments/{id}
  • PATCH /payments/{id}
  • POST /payments/{id}/capture
  • POST /refunds
  • GET /refunds/{id}

Create a new payment intent. The payment is initially created in pending state and must be explicitly captured using the POST /payments/{'{id}'}/capture endpoint, or set "capture_immediately": true to capture in a single call.

# Create a payment intent
curl -X POST https://api.example.com/v2/payments \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 4999,
    "currency": "USD",
    "customer_id": "cus_01HXYZ123",
    "description": "Order #10042 — 3x Widget Pro",
    "capture_immediately": false,
    "metadata": {
      "order_id": "ord_01HXYZ999"
    }
  }'

# Example response (201 Created)
{
  "id": "pay_01HABC456",
  "status": "pending",
  "amount": 4999,
  "currency": "USD",
  "created_at": "2026-05-13T16:30:00Z"
}

Notes

  • amount is always in the smallest currency unit (cents for USD)
  • Payments expire after 30 minutes if not captured
  • Rate limit: 200 requests/minute per API key
