Namespaces & scopes

Core system types live under the shared system root and are available to every tenant. Tenants also have private namespaces.

System root (core)

std/* — core system types curated by the platform; visible to all tenants; immutable except by platform governance.

# base URI
api.objectified.dev/types/std/

Tenant namespaces

tenant/<slug>/* — each tenant's private types; can reference core types but are isolated from other tenants.

# base URI
api.objectified.dev/types/tenant/<slug>/

Scope precedence & resolution order

When a property's type lookup resolves a bare name or relative $ref, namespaces are searched in this order:

  1. 1

    Tenant namespace tenant/<slug>/…

    Most specific — the tenant's own types take precedence.

  2. 2

    Imported vendor namespaces

    Vendor schemas imported for that tenant, e.g. vendor/fhir/r4.

  3. 3

    System core std/…

    Shared fallback available to every tenant.

Core types may be referenced by tenant types; tenant types are never visible across tenants or from core.

Promote to core

Promote a vetted tenant type into the system root std/* so all tenants can reference it. Promotion requires platform governance review.

# promotion example
tenant/acme/v1/types/money
  ↓ promote
std/v0/types/money # visible to all tenants
requires platform admin approval
